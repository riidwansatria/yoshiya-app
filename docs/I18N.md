# Internationalisation (i18n)

The app supports English (`en`) and Japanese (`ja`) via `next-intl`. Japanese is the default.

## How it is wired up

**Locale resolution** — `i18n/request.ts` reads the `NEXT_LOCALE` cookie on every server request. If the cookie value is `en` or `ja` it is used; otherwise the locale falls back to `ja`.

**Root layout** — `app/layout.tsx` calls `getLocale()` and `getMessages()` from `next-intl/server`, sets `<html lang={locale}>`, and wraps the tree in `<NextIntlClientProvider messages={messages}>` so Client Components can access translations without an extra network round-trip.

**Next.js plugin** — `next.config.ts` wraps the config with `createNextIntlPlugin()`, which handles the middleware and request integration automatically.

## Locale files

```
messages/
  en.json   # English strings
  ja.json   # Japanese strings
```

Strings are nested by feature area (e.g. `nav`, `settings`, `kitchen`). Both files must stay in sync — every key present in `en.json` must exist in `ja.json` and vice versa.

## Using translations in Server Components

```ts
import { getTranslations } from 'next-intl/server';

export default async function Page() {
    const t = await getTranslations('nav');
    return <h1>{t('kitchen')}</h1>;
}
```

## Using translations in Client Components

```ts
'use client';
import { useTranslations } from 'next-intl';

export function MyComponent() {
    const t = useTranslations('nav');
    return <span>{t('kitchen')}</span>;
}
```

`useTranslations` works in Client Components because `NextIntlClientProvider` passes all messages down from the root layout. No additional setup is needed per component.

## Adding new strings

1. Add the key to `messages/en.json` under the appropriate namespace.
2. Add the same key to `messages/ja.json` with the Japanese translation.
3. Use `getTranslations` (server) or `useTranslations` (client) with the matching namespace.

Never hardcode English strings directly in components — always go through the translation layer so Japanese users see the correct text.

## Changing the locale

The locale is stored in the `NEXT_LOCALE` cookie. The user menu in the dashboard includes a language toggle that sets this cookie and reloads the page. The `LocaleDefault` component (`components/locale-default.tsx`) handles setting the cookie to the default locale on first visit.
